# habr-corpus-parser

Парсер статей и комментариев Хабра в markdown-корпус. Собран в Claude Code (Opus 4.8) в рамках полевого эксперимента с [библиотекой промптов Claude Code](https://code.claude.com/docs/en/prompt-library): пять промптов по паттернам библиотеки, каждый — отдельная сессия, история коммитов повторяет хронологию прогонов. Подробности — в статье на Хабре [ссылка на статью](https://habr.com/ru/articles/1055714/).

## Что умеет

- `urls <file>` — спарсить статьи по списку URL (см. `urls.md`)
- `author <username>` — все публикации автора через listing API
- `reindex` — пересобрать INDEX.md и corpus.jsonl
- `validate` — проверка полноты корпуса: ноль пустых обязательных полей или явный лог

## Быстрый старт

```bash
python3 -m venv venv
venv/bin/pip install markdownify
venv/bin/python -m habr_parser urls urls.md
venv/bin/python -m habr_parser author VitTurov
```

Конфиг — `habr.yaml`: свой alias (статьи автора идут в `corpus/mine/`, остальные — в `corpus/reference/`), задержка между запросами, User-Agent.

К Хабру парсер обращается вежливо: пауза 2,5 секунды между запросами, внятный User-Agent, ретраи с экспоненциальным бэкоффом. Результаты (`corpus/`) в репозиторий не коммитятся — тексты статей и комментарии читателей публикации не подлежат.

## Про SPEC.md

Спецификация — артефакт первого прогона эксперимента (промпт-интервью) и намеренно оставлена как есть, со всеми следами: в ней выдуманный username и ошибочное утверждение о недоступности закладок. Оба следа разобраны в статье. Не исправляйте их через PR — это музей.
